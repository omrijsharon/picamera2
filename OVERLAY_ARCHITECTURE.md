## Comparison: Naive vs. OverlayHelper

### Naive Approach (DON'T DO THIS)
```
Every frame:
  1. overlay = np.zeros((h, w, 4))     â† Allocate 1.2 MB
  2. Draw all elements                  â† Write entire buffer
  3. picam2.set_overlay(overlay)       â† QImage copies buffer
  4. GC deallocates old array           â† Cleanup overhead

Memory operations: 1.2 MB allocation + 1.2 MB write + potential copy
Time: ~0.3-0.8ms
Memory churn: 72-480 MB/s at 60 FPS
```

### OverlayHelper Approach (OPTIMAL)
```
Initialization (once):
  1. overlay = np.zeros((h, w, 4))     â† Allocate 1.2 MB ONCE

Every frame:
  1. overlay[dirty_regions] = 0        â† Clear ~1-5 KB (changed areas only)
  2. Draw changed elements              â† Write ~1-5 KB
  3. array = overlay.get_array()       â† Return pointer (no copy)
  4. picam2.set_overlay(array)         â† QImage wraps buffer (no copy)

Memory operations: ~1-5 KB write (200-1000x less!)
Time: <0.05ms (5-15x faster)
Memory churn: 0 (no allocations)
```

## Key Optimizations

### 1. Pre-Allocation
```python
# Bad: Allocates every frame
overlay = np.zeros((480, 640, 4))

# Good: Allocate once, reuse forever
self.overlay = np.zeros((480, 640, 4))  # In __init__
```

### 2. Partial Updates
```python
# Bad: Clear entire frame
overlay[:] = 0

# Good: Only clear changed regions
for x, y, w, h in dirty_regions:
    overlay[y:y+h, x:x+w] = 0
```

### 3. Element Tracking
```python
# Track what's where
self.elements = {
    "battery": OverlayElement(x=10, y=10, w=100, h=20),
}

# When updating, mark old region as dirty
old = self.elements["battery"]
self.dirty_regions.append(old.get_bounds())
```

### 4. Zero-Copy Access
```python
# Returns pointer, not copy
def get_array(self):
    return self.overlay  # No np.copy()!
```

### 5. C-Contiguous Layout
```python
# Matches QImage memory layout (no conversion needed)
np.zeros((h, w, 4), dtype=uint8, order='C')
#                                 ^^^^^^^^
#                                 C-contiguous = QImage-friendly
```
